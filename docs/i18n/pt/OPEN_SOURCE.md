## Licenças/Decisão de licença

O projeto usa a licença **Apache License 2.0** (`SPDX: Apache-2.0`). Esse
licença permissiva: permite uso e modificação privada, comercial e aberta
e distribuição, ao mesmo tempo que exige a preservação da licença e atribuição, conforme alterado
arquivos e contém uma licença expressa de patente.

O projeto usa a **Licença Apache 2.0** (`SPDX: Apache-2.0`). Permite privacidade,
uso, modificação e distribuição comercial e de código aberto, ao mesmo tempo que exige
preservação de licença/atribuição, avisos para arquivos alterados e fornecimento de um
concessão explícita de patente.

Por que não o MIT? O MIT é mais curto e também permissivo, mas não inclui o Apache-2.0-
uma provisão de patente expressa e detalhada semelhante a Um Java integrável
para a biblioteca cliente, o Apache-2.0 fornece um contrato empresarial mais limpo.

Por que não o MIT? O MIT é mais curto e permissivo, mas não contém o mesmo conteúdo explícito,
termos detalhados de patente. O Apache-2.0 oferece aos adotantes de uma biblioteca Java reutilizável uma visão mais clara
contrato de licença voltado para a empresa.

O projeto não tem licença dupla. O usuário não precisa alternar entre MIT e Apache
para escolher: uma única licença, Apache-2.0 é válida.

Este não é um projeto com licença dupla. Apache-2.0 é a única licença do projeto.

## Verificação de direitos antes de publicar

Antes de tornar o repo público, o proprietário deve confirmar que:

- direito de liberar todo o código-fonte e documentação no Apache-2.0;
- o código não é propriedade do empregador ou do cliente;
- não contém códigos, segredos, dados pessoais ou de clientes copiados sem autorização;
- a origem e a licença de todas as principais peças de terceiros estão documentadas.

Antes de tornar o repositório público, o proprietário deve confirmar o direito de licenciar todos
código e documentação, a propriedade do empregador/cliente não se aplica, nenhum código copiado,
segredos, dados pessoais ou dados de clientes estão presentes e material de terceiros é
devidamente atribuído.
